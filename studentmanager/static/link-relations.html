StudentManager API: Link Relations

Standard link relations

  • self: carried by all resources returned by the API. Points to the resource itself. It is also carried by all Items in Collections, pointing at the related resource.
  • profile: points to the related proflie of the current resource.
  • collection: carried by Item resources, points to the related Collection.
  • edit: carried by Item resources, points to itself, specifying PUT method and the appropriate JSON schema to use for the request

Custom link relations

All the following relations are defined in the 'studman' namespace, prepended to the relation and separated by a colon (e.g. 'studman:courses-all').
  • students-all, courses-all, assessments: this links point to the three main resources in our API: StudentCollection, CourseCollection and AssessmentCollection.
  • add-student, add-course, add-assessment: each one of the aforementioned collections wlil point to itself with the corresponding control, specifying POST method and the appropriate JSON schema to use for the request.
  • student-assessments, course-assessments: each StudentItem and CourseItem wlil carry a link to the respective AssessmentCollection.
  • student, course: each AssessmentItem wlil point to its related Student and Course through these links. In addition, StudentAssessmentCollection (and CourseAssessmentCollection) resources wlil point to the related StudentItem (and CourseItem). (The student link is also used from the ProfilePictureItem to the corresponding StudentIten)
  • delete: a utliity link carried by every Item resource pointing itself, specyfing DELETE method.
  • propic: a link from a StudentItem to the corresponding ProfilePictureItem.
